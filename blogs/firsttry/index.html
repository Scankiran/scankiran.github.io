Zero to Hugo: Build Websites With Nocode

Said Çankıran | May 23, 2024 min read

In this article, I’ll write about how to create your personal website or landing page or any website which you want to show to internet with Hugo and Github Pages. And ofcourse with nocode.

No matter what field you work in, with a little bit of Git and Terminal knowladge, you can easily set up and deploy the site you want. If you’re ready, let’s start.

/content/zeroToHugo/hugoCover.png

Öncelikle Hugo nedir? Hugo bir statik site oluşturucusudur. Hazır temaların üzerinde config dosyalarını düzenleyerek kendi konfigurasyonlarınız ile site oluşturmanızı sağlar. Yani aslında bir text dosyasını düzenleyerek kişisel websitenizi oluşturabilirsiniz. Yüzlerce farklı hazır tema kullanabilirsiniz. Bu temaların arasında Portfolyo, Personal Blog, Landing page gibi farklı kategorilerde yüzlerce örnek bulabilirsiniz.

Peki neden Hugo? Hugo hızlı çalışır ve hafiftir. Siteniz ne kadar büyük olursa olsun hızlıca build olur ve sizin oluşturduğunuz statik contentlerde websitesini bir kaç saniyede oluşturur. Örnek vermek gerekirse, 60 blog yazılı ve her yazının içinde 2-3 adet resim bulunan siteyi ~5 saniye gibi kısa bir sürede oluşturup yüklemeye hazır hale getirmektedir. Bununla birlikte az önce bahsettiğim gibi çok fazla tema alternatifi var. Açık kaynaklı. Dilerseniz siz de katkıda yapabilirsiniz. Ve bence en önemlisi dökümantasyonu detaylı bir şekilde anlatıyor. Ve soru cevap sitesi var. Herhangi bir problem yaşadığınızda sizden önce de bu problemi yaşayan olabildiği için hızlıca çözüme ulaştırabiliyor.

Bu kadar giriş yeter. Hadi o zaman sitemizi oluşturmaya başlayalım. İhtiyacımız olan şeyler, text editor (sublime etc.), terminal, Github Account.

Öncelikle bilgisayarımıza Hugoyu kurmakla başlayalım. Ben Mac’e sahip olduğum için Mac kurulumundan devam edeceğim. Farklı bir işletim sistemine sahipseniz Hugo Install bakmanızda fayda var.

Brew kullanarak kolayca Hugoyu yüklüyorum.

brew install hugo

Hugo’yu kurduktan sonra cihazınıza Git’i de kurmanız gerekli. Git’i de yine Brew ile yüklüyorum.

brew install git

İşte kurulum bu kadar. Artık sitemizi oluşturmaya başlayabiliriz.

Sitemizin dosyalarını saklamak istediğimiz yere giderek yeni bir terminal sekmesi oluşturalım.

hugo new site personal_website
cd personal_website
git init

Bu 3 komut ile sitemizin temelini oluşturalım. Peki bu 3 komut ne yapıyor?

  1. satırdaki komut ile hugo’dan kendi konfigurasyonlarını kullnarak bir site temeli oluşturuyor. Oluşturduğu dosyaları gidersek ileride sıkça kullanacağımız content, static gibi klasörleri bizim için oluşturduğunu göreceğiz.
  2. satırdaki komut ise oluşturulan klasörün detayına gidiyor.
  3. satırdaki komut ise git’i websitemizin temel klasörüne initialize ediyor.

Sıra geldi beğendiğimiz temayı entegre etmeye. Ben Hugo Themes adresine gidip beğendiğim bir temayı ekleyeceğim. Tema olarak PaperMod‘u seçtim.

git submodule add --depth=1 https://github.com/adityatelange/hugo-PaperMod.git themes/PaperMod

komutu ile PaperMod temasını klasörümüze clone’lanmasını sağlıyoruz.

PaperMod yaml konfigurasyon dosyasını kullanıyor. Bu yüzden biz de aşağıdaki komut ile config.yaml dosyası oluşturuyoruz. (Temadan temaya değişebilir. Bazı temalar toml bazı demalar yaml türünü kullanabilir. Temanın kurulum adresinde ya da örnek sitelerinde bu bilgi bulunur.)

touch config.yaml

yaml dosyasını ekledikten sonra temanın bize vermiş olduğu örnek yaml dosyasının içeriğini local’imizdeki boş yaml dosyasının içerisine kopyalıyorum.

Ve sıra geldi local’imizde sitemizi görmeye. Sitemizi localhost’ta ayağa kaldırmak için hugo server komutunu kullanacağız. Bu komut bizim içeride eklediğimiz content, resim, yazıları ve temamızı kullnarak bizim için statik websitesi oluşturacak ve localhostumuzda deploy edecek. Komutu çalıştırdığımızda şu şekilde local adresimizi verdiği çıktıyla karşılaşmamız gerek.

// Resim

Ve ta-da! Localhost’a gittiğimizde şu şekilde bir sayfa bizi karşılıyor olacak.

// resim

Bu komut hotreload mekanizması ile çalışıyor. Yani siz yaml dosyasını editleyip kaydettiğiniz anda değişikliklerinizin tamamı otomatik olarak yansıyacaktır.

Bundan sonrası biraz sizin içeriği düzenlemeniz gerekiyor. Mesela ben yaml dosyası içerisindeki .. parametresini .. olarak değiştiriyorum ve anında yansıyarak sitem şu şekilde güncelleniyor. yaml dosyasında resim eklemeniz gerken parametreler için sitemizin bulunduğu klasörde static klasörüne gidip oraya resimleri eklemeniz ve yeniden hugo server diyerek resimleri tanımasını sağlamalısınız. Daha sonra parametreye static/img/favicon.png şeklinde path vererek resimleri gösterebilirsiniz. Buradaki parametrelerin açıklamaları PaperMod‘un sayfasında var. Dilerseniz kendiniz değişkenleri değiştirip kurcalayarakta düzenleyebilirsiniz.

Sitemizi genel hatlarıyla düzenledik. Sıra geldi Blog yazısı kısmına.

Öncelikle burada dikkat etmemiz gereken bir konu var. Temamızın blog temasını desteklemesi gerekiyor. Bu şekilde yazdığımız blog yazılarını sitemizde gösterebiliriz. Peki desteklemiyorsa ne yapabiliriz? Farklı bir site projesi oluşturarak blog yazısı destekleyen bir tema seçerek blog’umuzu oluşturabiliriz. Daha sonra kişisel sitemizden blog sitemize yönlendirme yapabiliriz.

Yazımızı oluşturken

hugo new content posts/my-first-post.md

komutunu kullanarak yapıya uygun şekilde markdown dosyası oluşturur. Geriye size sadece ilgili markdown doyasını açıp düzenlemesi kalır. Komutu çalıştırdıktan sonra örnek markdown dosyasımızın içerisi bu şekilde.

+++ title = ‘My First Post’ date = 2024-05-23T21:17:11+03:00 draft = true +++

Burada blog yazımız draft yani taslak olarak işaretlendiği için yazımız sitemizde görünmeye bilir. hugo server -D ya da hugo server --buildDrafts komutları ile taslak dosyaların da build edilmesini sağlayabilirsiniz.

Markdown zengin formatları destekleyen bir dosya ve yazı türüdür. Sizin okuduğunuz bu yazı da bir markdown dosyası :) Bu yazıdaki resimler hariç diğer her şey markdown formatları kullanılarak oluşturuldu. Matematik formulleri, kodlar, alıntılar, açılıp kapanabilen başlıklar, listeler vb aklınıza gelebilecek her türlü formatı destekler.

Markdown ile ilgili daha fazla bilgiyi ve cheatsheet’i buraya bırakıyorum. İncelemenizi tavsiye ederim.

Blog yazımızı yazdık. Sıra geldi sitemizi deploy etmeye.

Öncelikle sitemizi build etmeden önce kullanacağımız adresi config dosyamızda baseURLparametresini güncelleyelim. Mesela benimki https://scankiran.github.io

BaseURL’imizi tanımladıktan sonra sitemizi hugo ile build edelim. Bu sefer localhost yerine bir adrese göndereceğimiz için bizim build dosyalarının çıktısını almamız gerekiyor. Bunun için sadece hugo komutunu kullanacağız. Bu komut bizim yazdığımız yazılardan ve oluşturduğumuz config dosyasından bize statik site çıktısı üretecek ve bunu public klasörü altına toplayacak. // resim

Bizim bundan sonra yapmamız gereken ise bu public klasörünü Github’a yüklemek olacak.

Öncelikle bunun için bize gereken Github’da kullanıcıadımız+github.io ile açılmış bir public repository. Mesela benimki scankiran.github.io.

Repository oluşturduktan sonra burada iki farklı şekilde ilerleyebiliriz. Dilersek public klasörünü direkt upload files kısmından yükleyebiliriz. // resim

Ya da diğer seçeneğimiz klasik git komutları ile pushlayarak public klasörünü bu repository’e pushlayabiliriz. Benim tercihim bu yönde olacaktır.

Dosyalarımızı pushladıktan sonra repository’imizin ayarlarından pages sekmesine gidip Deploy From Branch - Main ayarlarını yapalım.

// resim

Bunları yaptıktan 1-2 dakika sonra sitemiz Github Actions tarafından build edilip deploy edilecek. İşlemleri Actions sekmesinden de kontrol edebiliriz.

Aynı zamanda işlem sonucu ana sayfada commit mesajımızın yanında yeşil tik işareti ile de görüntülenebilir. Bu yazımda sizlerle birlikte sıfırdan Hugo ve Github Pages ile birlikte kişisel web sitemizi hazırladık. Umarım sizlere faydalı olmuştur.

Bir sonraki yazılarda görüşmek dileğiyle.
